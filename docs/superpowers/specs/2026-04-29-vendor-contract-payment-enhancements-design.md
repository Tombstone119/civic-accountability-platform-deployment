## Overview

## Wave 1 ‚Äî Bug Fixes & Enhancements (no new dependencies)

### 1. Remove from Blacklist Fix

**Problem:** The "Remove from Blacklist" button in `VendorDetailPage.tsx` calls the backend but does not update local React state on success, so the UI does not reflect the change.

**Fix:** In the button handler, after a successful `DELETE /api/vendors/:id/blacklist` call, update local state:
```ts
setVendor(v => v ? { ...v, isBlacklisted: false, blacklistReason: undefined, blacklistedAt: undefined } : v)
```
No backend changes needed ‚Äî `vendorController.unblacklist` is fully implemented.

**Files changed:**
- `client/src/pages/vendors/VendorDetailPage.tsx` ‚Äî fix handler + add `vendorService.removeFromBlacklist`
- `client/src/services/vendorService.ts` ‚Äî add `removeFromBlacklist(id)` method

---

### 2. Edit Contract

**Problem:** `ContractDetailPage.tsx` has a `{/* edit modal future */}` placeholder. The Edit button does nothing. The `PUT /api/contracts/:id` endpoint is fully implemented.

**Fix:** Add an edit drawer (same pattern as VendorDetailPage edit drawer) with fields:
- title, description, status, procurementMethod, startDate, endDate, contractValue

On save: call `contractService.update(id, form)`, update `contract` state in place.

**Files changed:**
- `client/src/pages/contracts/ContractDetailPage.tsx` ‚Äî add drawer state + form + handler

---

### 3. Contract PDF Report Download

**Problem:** "Download Report" button has no `onClick` handler.

**Backend:** New endpoint `GET /api/contracts/:id/report`.
- Auth-protected (any authenticated user)
- Uses `pdfkit` to generate PDF in memory and stream directly to response
- `Content-Type: application/pdf`
- `Content-Disposition: attachment; filename="contract-{contractNumber}-report.pdf"`

**Report sections:**
1. Header: "VeriTrack ‚Äî Contract Report" + generation date
2. Contract summary: number, title, vendor, department, method, status, value, paid, remaining, dates
3. Line Items table: description, qty, unit, unit price, total, overpricing flag
4. Payment History table: ref number, amount, date, method, status
5. Footer: "Generated by VeriTrack ‚Äî Confidential"

**Client:** New `contractService.downloadReport(id)` ‚Äî GET with `responseType: 'blob'`, creates object URL, triggers download via temporary `<a>` element. Button shows loading state during request.

**New dependency:** `pdfkit`, `@types/pdfkit` on server.

**Files changed:**
- `server/src/controllers/contractController.ts` ‚Äî add `downloadReport` method
- `server/src/routes/contractRoutes.ts` ‚Äî register `GET /:id/report` before `/:id`
- `client/src/services/contractService.ts` ‚Äî add `downloadReport(id)`
- `client/src/pages/contracts/ContractDetailPage.tsx` ‚Äî wire button onClick

---

### 4. Payment Edit + Delete in ContractDetailPage

**Problem:** Payment History table has an Actions column with a `MoreVertical` icon but no handler.

**Fix:** Replace icon with two explicit buttons ‚Äî Edit and Delete.

- **Edit:** Opens a small drawer with fields: status, notes, referenceNumber. Calls `PUT /api/payments/:id`.
- **Delete:** Confirmation modal. Calls `DELETE /api/payments/:id`. Blocked on server if status is `completed`.

After each action, re-fetch payments list and update `payments` state.

**Files changed:**
- `client/src/pages/contracts/ContractDetailPage.tsx` ‚Äî add edit drawer + delete modal + handlers

---

### 5. Payment Status Update for Procurement Officers

**Problem:** `PUT /api/payments/:id` is restricted to `admin` only. Procurement officers cannot update payment status from `PaymentsListPage.tsx`.

**Backend change:** In `paymentRoutes.ts`, change `requireAdmin` to `requireOfficer` on `PUT /api/payments/:id`.

**Frontend:** In `PaymentsListPage.tsx`, add a status update action per row (visible to `admin` and `procurement_officer` roles). Inline dropdown or "Update Status" button opening a small modal. On confirm, call `paymentService.update(id, {status})` and refresh the list.

**Files changed:**
- `server/src/routes/paymentRoutes.ts` ‚Äî change middleware on PUT route
- `client/src/pages/payments/PaymentsListPage.tsx` ‚Äî add status update UI (role-gated)

---

### 6. Spending Summary "Refresh Data" Fix

**Problem:** The "Refresh Data" button calls `publicService.refreshSpending()` but `publicService` may not use the authenticated Axios instance, causing the `POST /api/spending/refresh-summary` (admin-only) call to fail silently or return 401.

**Fix:** Ensure `refreshSpending()` uses the authenticated `api` instance (not a plain axios call). Handle the `200` response by showing a success alert and re-fetching summaries. Handle errors with an error alert. The backend endpoint is fully implemented.

**Files changed:**
- `client/src/services/publicService.ts` ‚Äî fix `refreshSpending` to use authenticated `api` instance
- `client/src/pages/spending/SpendingSummaryPage.tsx` ‚Äî fix success/error handling on refresh

## Wave 2 ‚Äî GridFS File Upload System

### 7. GridFS Setup

**New file:** `server/src/config/gridfs.ts`
- Exports `getGridFSBucket()` ‚Äî lazily initializes a `mongoose.mongo.GridFSBucket` on the `uploads` bucket
- Called after mongoose connection is established

**New dependencies:** `multer`, `@types/multer` on server. No `multer-gridfs-storage` needed ‚Äî we use `memoryStorage()` and upload the buffer directly via the GridFS bucket API.

---

### 8. Upload Endpoint (`POST /api/vendors/:id/documents`)

**Replaces** the existing metadata-only `addDocument` endpoint with a file-aware version.

- Multer configured with `memoryStorage()` ‚Äî file bytes in `req.file.buffer`
- Accepted MIME types: `application/pdf`, `image/*`, `application/msword`, `application/vnd.openxmlformats-officedocument.wordprocessingml.document`
- Max file size: 10MB (enforced by multer)
- After upload to GridFS, creates `VendorDocument` with `fileUrl` = `"gridfs:<ObjectId>"`
- Form fields: `documentType`, `documentNumber`, `issueDate`, `expiryDate`, plus `file`

**Files changed:**
- `server/src/controllers/vendorController.ts` ‚Äî update `addDocument` to handle `req.file`
- `server/src/routes/vendorRoutes.ts` ‚Äî add multer middleware to upload route

---

### 9. File Download Endpoint (`GET /api/vendors/:id/documents/:docId/file`)

- Auth-protected via `authMiddleware` + `requireAuthenticated`
- Looks up `VendorDocument` by `docId`, extracts GridFS ObjectId from `fileUrl`
- Opens `GridFSBucket.openDownloadStream(objectId)` and pipes to response
- Sets `Content-Type` from stored MIME type, `Content-Disposition: inline; filename="<originalName>"`
- Returns 404 if document or file not found

**Files changed:**
- `server/src/controllers/vendorController.ts` ‚Äî add `downloadDocumentFile` method
- `server/src/routes/vendorRoutes.ts` ‚Äî register `GET /:id/documents/:docId/file`

---

### 10. VendorDocument Model Update

Add fields to `VendorDocument` schema (all optional, backward-compatible):
- `gridfsId: String` ‚Äî the raw GridFS ObjectId string
- `originalName: String` ‚Äî original filename from upload
- `mimeType: String` ‚Äî detected MIME type
- `fileSize: Number` ‚Äî bytes

Existing `fileUrl` field is repurposed to store `"gridfs:<objectId>"` as a convention.

**Files changed:**
- `server/src/models/VendorDocument.ts` ‚Äî add optional fields

---

### 11. Frontend Upload UI

**In `VendorDetailPage.tsx`:**
- Replace metadata-only Upload Document drawer with a file picker form
- `<input type="file" accept=".pdf,.jpg,.jpeg,.png,.doc,.docx">` replaces the `fileUrl` text input
- On submit: build `FormData`, call `vendorService.uploadDocument(id, formData)`
- Show upload progress via Axios `onUploadProgress` callback (simple percentage display)
- On success: prepend new document to `documents` state

**Files changed:**
- `client/src/services/vendorService.ts` ‚Äî update `uploadDocument` to post `FormData`
- `client/src/pages/vendors/VendorDetailPage.tsx` ‚Äî update upload form

---

### 12. Documents Table Actions

Add three action buttons per row in the documents table:

**View:**
- Calls `GET /api/vendors/:id/documents/:docId/file` via authenticated Axios with `responseType: 'blob'`
- Creates a temporary object URL, opens in new tab via `window.open()`
- Shows loading state on button during fetch

**Edit:**
- Opens inline drawer with: `documentNumber`, `issueDate`, `expiryDate`, `isVerified` (admin only)
- Calls `PUT /api/vendors/:id/documents/:docId`
- Updates row in local `documents` state on success

**Delete:**
- Confirmation modal: "Are you sure you want to delete this document?"
- Calls `DELETE /api/vendors/:id/documents/:docId`
- Removes row from local `documents` state on success
- Admin-only (button hidden for non-admins)

**Files changed:**
- `client/src/pages/vendors/VendorDetailPage.tsx` ‚Äî add View/Edit/Delete columns + handlers
- `client/src/services/vendorService.ts` ‚Äî add `viewDocumentFile`, `updateDocument`, `deleteDocument` methods

## Error Handling

- All async handlers wrapped in try/catch; errors shown via existing Alert component pattern
- Multer file size/type errors returned as 400 with descriptive message
- GridFS stream errors caught and returned as 500
- Client shows inline error messages under forms (existing pattern)
- Loading states on all async actions (existing pattern: `saving`, `uploading`, `deleting` booleans)
